初用hexo
Jul 21 2022

这两天使用hexo搭建了自己的Blog, 总结一下容易遇到的问题以及分享一些好的文章

准备工作

  1. node.js
  2. hexo
    npm install hexo-cli -g
    hexo官方地址
  3. 准备好自己喜欢的主题(可以不配置主题)
    hexo主题地址

正式开始

  1. 使用hexo建站

    hexo init <folder>
cd <folder>
npm install

    hexo站点目录结构

    .
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

  2. 配置主题(可以省略)

    1. 首先将git上获取到的主题文件夹换一个名字(也可以选择不换)
2. 将主题文件夹移动到 hexo站根目录下的themes
3. 修改hexo站根目录下的_config.yml
    修改 theme: <主题文件夹名>

  3. 本地运行
    hexo s

  4. 创建新文章
    hexo n "文章名" / hexo new "文章名"

    hexo 创建的文章都存在于 hexo站点/source/_posts/

  5. 将hexo站点部署到github上
    首先得在自己的github上有一个<自己githu名称>.github.io的仓库
    hexo部署主要有两种方式 自动部署和手动部署，这里只讲自动部署

    1. 安装hexo部署github插件
      npm install hexo-deployer-git --save
    2. 修改hexo站根目录下的_config
      deploy:
    type: git
    repo: <自己githu名称>.github.io的仓库地址
    branch: main(更新到哪个分支，建议使用main)
    message: 更新说明
    token: github koken密钥

      github koken密钥获取方式
      打开github的settings -> Developer settings -> Personal access tokens -> Generate new token

    3. 正式部署
      配置修改好之后在hexo站点根目录下运行
      hexo clean #清除缓存 网页正常情况下可以忽略此条命令
hexo g #生成静态网页
hexo d #开始部署

  6. 关于修改主题的配置

    主题的配置都存在于 hexo站点/themes/<主题名>/_config.yml
